Send MessageWhen AI generates User Interfaces (UI) or frontend code, it often leaves behind specific "fingerprints"‚Äîsubtle (and sometimes obvious) mistakes that act as dead giveaways. These typically fall into two categories: **Visual/Design Flaws** and **Code Quality Issues**.

Here are the most common mistakes that make a project *feel* AI-generated:

### 1. The "Happy Path" Bias (The #1 Giveaway)
AI is trained to solve the immediate problem you ask about, assuming everything goes perfectly. It rarely accounts for real-world messiness.
*   **In UI Design:** The interface looks perfect with ideal data (e.g., short names like "John Doe"), but breaks completely when real content is dropped in (e.g., a long German last name overlaps the delete button).
*   **In Code:** It writes code that works *only* if the user does exactly what is expected. It often forgets **error handling**, **loading states**, or **empty states**.
    *   *The "Tell":* You click a button and nothing happens because the API failed, but the AI didn't code an error message to tell you why.

### 2. The "Uncanny Valley" of Details
AI often hallucinates details that look plausible at a glance but make no logical sense upon inspection.
*   **Nonsensical Text:** Instead of real copy, it generates weird, alien-like gibberish or misspells common words (e.g., "Logni" instead of "Login").
*   **Impossible Icons:** It might invent icons that don't exist in standard libraries, or create icons that are visually inconsistent (e.g., one icon is thick and cartoony, the next is thin and detailed).
*   **Phantom Libraries (in Code):** AI frequently imports libraries that don't exist or tries to use CSS properties that aren't real, simply because they *sound* like they should exist.

### 3. The "Generic Corporate Memphis" Aesthetic
AI models are trained on millions of popular Dribbble and Behance shots, leading to a very specific, over-polished, and soulless look.
*   **Over-Rounded Everything:** Excessive border-radius on every card and button.
*   **Vibrant but Meaningless Gradients:** Using blue/purple gradients everywhere without any brand logic.
*   **Lack of Hierarchy:** Everything looks "pretty," but it's hard to tell what the primary action is. The "Cancel" button often looks just as important as the "Confirm" button.

### 4. Accessibility Failures
Because AI prioritizes aesthetics over usability rules, it frequently generates designs that are hostile to users with disabilities.
*   **Low Contrast:** Light gray text on a white background (looks clean, but is unreadable).
*   **Tiny Click Targets:** Buttons that are too small for a human thumb to press reliably.
*   **Div Soup (in Code):** Using `<div>` for everything instead of semantic HTML tags like `<button>`, `<nav>`, or `<article>`, breaking screen readers.

### 5. Over-Engineered Spaghetti Code
When you look at the code, it often feels like it was written by a junior developer trying to impress a senior.
*   **Bloat:** Writing 50 lines of complex React/Vue logic for something that could have been 5 lines of CSS.
*   **Redundant Comments:** AI loves to comment on the obvious.
    *   *Example:* `// This function adds two numbers` right above `function add(a, b)`.
*   **Mixing Styles:** You might see three different ways of styling (Tailwind, inline styles, and CSS modules) all in the same file because the AI pulled from different training sources.
